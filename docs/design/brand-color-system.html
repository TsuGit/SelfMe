Brand Standard

SelfMe Brand Color System

这份文档定义 SelfMe 的统一配色制度。官网、文档、CLI、桌面端都必须从这一套颜色系统取值， 不允许各自新增主色、临时换蓝、或者随页面自由漂移。

核心原则： SelfMe 的品牌主线固定为 深海军蓝 背景体系和 冷青蓝 强调体系。不同载体只能调整使用强度，不能改掉品牌色相方向。
Principles

品牌原则

  • 品牌气质关键词固定为：冷静、现代、可信、深海感、未来感。
  • 主背景永远走深蓝黑体系，不走纯黑，也不走偏紫。
  • 主强调永远走冷青蓝体系，不允许每个页面换一种蓝。
  • 辅助色可以存在，但只能做少量语义区分或信息分层，不能抢主品牌色。
  • 官网可以更亮，文档可以更稳，CLI 可以更克制，但都必须使用同一套 token。

Usage

使用规则

Token 用途 禁止事项
accent-primary 品牌标题高亮、主按钮、链接 hover、关键选中态 不能大面积铺整个页面背景
accent-secondary 渐变辅助、动态图形、弱一级高亮 不能替代主品牌色独立使用为主色
accent-tertiary 成功标签、辅助类别标签 不能进入官网或 CLI 的主视觉第一层
accent-warm 少量强调、分类差异化 不能与主品牌蓝平级争抢主视觉
text-muted 说明文字、次级元信息、CLI 次级行 不能用于高优先级正文或 CTA
line-subtle 卡片边框、代码块边框、分隔线 不能亮到像强调色
Surfaces

载体映射

官网

允许使用更亮的 accent-primary 渐变和轻发光，但背景基底仍然必须来自 bg-base 系列。

文档

优先可读性和层次，强调色只做标题、tag、highlight，不做大面积情绪化发光。

CLI

只允许少量 accent。主体以 text-primary 和 text-muted 为主，强调色只用于 logo、关键状态或少量提示。

桌面端

介于官网和文档之间。比 CLI 丰富，但不能回到官网首页那种强营销张力。

Status

当前统一状态

官网

  • 官网已经统一到 bg-base、accent-primary、accent-secondary 这套标准值上。
  • 官网仍然保留比文档更亮、更有张力的使用方式，但底层颜色系统已经与文档一致。

文档

  • 品牌制度文档和 roadmap 文档已经统一到同一套背景、文字和强调色体系。
  • 文档中的绿色和金色继续只作为辅助色存在，不承担主品牌识别。

CLI

  • CLI 已经接入同一套品牌逻辑：logo 使用 accent-primary，次级信息使用 text-muted。
  • CLI 的用色强度严格低于官网，符合“同体系、不同强度”的载体规则。
当前结论： 官网、文档、CLI 现在已经不再是各自漂移的配色，而是共享一套品牌 token，并按载体强度差异化使用。
Governance

后续执行规则

  1. 以后新增页面、组件、端形态时，只能从这套系统取值，不允许重新发明品牌蓝。
  2. 如果需要新增颜色，必须先更新这份制度文档，再更新具体载体实现。
  3. 桌面端未来应直接继承本制度，不允许单独派生一套“桌面端专属品牌色”。
  4. CLI 如果继续扩展状态色、提示色、审批色，也必须继续映射到本制度的语义 token。
