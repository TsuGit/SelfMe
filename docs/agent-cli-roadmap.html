Execution Plan

SelfMe Agent CLI Roadmap

当前路线已经收敛：只做一个单体 TypeScript CLI agent，把输入体验、工具执行、审批、安全边界和真实 coding task 闭环做扎实。先不讨论 web、gateway、多端共享后端，也不为未来产品形态提前堆复杂架构。

唯一目标： 把当前 CLI 做成一个真正可用、可持续迭代的 agent 基线，而不是继续扩张产品壳层。
Position

产品边界

  • 这是一个单进程、本地运行的 CLI agent。
  • 当前技术栈固定为 TypeScript，不再讨论 Rust / TypeScript 选型。
  • 重点是 terminal 交互、agent runtime、tool execution、provider integration、local storage。
  • 所有设计都优先服务“真实 coding work 能不能稳定完成”，不是服务多端扩张。
评估标准只有一个：它是否越来越像一个真正能工作的 agent，而不是一个会聊天的终端 UI。
Baseline

当前基线

Terminal

已具备稳定输入区、多行输入、真实光标、流式回复、底部菜单和基础状态展示。

Commands

当前保留最小命令面：/help、/tools、/stop、/read、/write、/edit、/shell。

Approvals

shell、write、edit 已接入审批流，批准与拒绝路径可用，正常提示不再污染消息区。

Storage

已保留 transcript、logs、settings 等必要存储，去掉了复杂恢复和多余状态层。

Runtime Control

已具备单活跃任务模型、忙时拦截新消息、Esc / Ctrl+C / /stop 中断当前任务，以及 provider / shell abort 贯通。

Principles

当前阶段的开发原则

  • 先补真实能力，再补展示层修饰。
  • 新功能必须进入统一 runtime 和 tool contract，不允许再长出第二套流程。
  • 临时提示、帮助、命令说明尽量走底部菜单或瞬时提示，不污染主消息流。
  • 所有高风险操作必须有清晰审批，不允许因为“先能用”而绕开。
  • 任何新增复杂度，都必须能直接提升真实任务完成率，否则不做。
Next

下一阶段要做什么

  1. 真实 coding task 回归

    不再只测单句命令，开始用一批真实小任务验证闭环，例如读文件、改文件、跑命令、看结果、继续修正，确认 agent 真能持续完成工作。

  2. 上下文裁剪

    transcript 全量保留，但传给模型的上下文必须裁剪，避免长会话持续退化。

  3. 工具展示收口

    继续收工具块展示、长输出样本、失败说明和最终回答的克制程度，避免信息重复或工具噪音过强。

  4. 最小命令面维持稳定

    命令系统只保留最小集，不再扩张，但要继续保证 /stop、审批、帮助、命令补全这些入口稳定可用。

Not Now

明确暂不做

  • web UI。
  • desktop app。
  • gateway / ws / 多端共享后端。
  • 复杂 session browser、history browser、search browser。
  • checkpoint、任务恢复系统、重型计划系统。
  • 为了未来产品化预埋的大而全壳层。
这些不是永远不做，而是当前阶段不允许抢占主线。主线只有一个：把 CLI agent 能力闭环做扎实。
Milestones

阶段里程碑

M1

读文件、跑 shell、审批流稳定；命令面干净；正常交互不污染 transcript。

M2

已补齐写文件和编辑文件工具，agent 可以在审批后真正修改项目内容。

M3

中断控制和单活跃任务模型完成；接下来补上下文裁剪和工具输出收口，保证长对话仍然稳定。

M4

通过一批真实 coding task 回归，确认它已经是可用 agent，而不是仅靠演示路径成立。

Definition

什么叫这一版已经成立

  • 用户能稳定输入、编辑、发送多行内容。
  • 模型回复稳定流式输出，工具调用反馈清晰。
  • agent 可以读文件、改文件、跑 shell，并通过审批安全执行。
  • 长一点的对话不会因为上下文和工具日志失控而明显退化。
  • 可以独立完成一类真实小型 coding task，而不是只会执行单点命令。
Immediate

当前建议的直接开工项

下一步直接做 真实 coding task 回归 + 上下文裁剪设计。基础自动工具调用、审批和中断控制已经成立， 现在该验证它在连续真实任务里的稳定性，并开始处理长会话上下文退化问题。
